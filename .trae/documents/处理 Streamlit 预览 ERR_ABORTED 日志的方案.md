1. 路径确认：在项目根目录中查找 `fix.app.py` 的实际位置；若不在根目录，定位其完整路径。
2. 进程清理：停止当前端口 `8510` 上的已运行应用，避免多实例导致预览重连与 ERR_ABORTED 日志。
3. 启动服务：
   - 若为 Streamlit 应用：运行 `python -m streamlit run fix.app.py --server.port 8512`。
   - 若为普通 Python 脚本：运行 `python fix.app.py`，再根据脚本输出确认端口/URL。
4. 依赖验证：若启动报错（如缺少库），根据日志安装或修复缺失依赖后重试；保留原环境不改动库版本。
5. 预览与验证：打开 `http://localhost:8512`（或脚本输出的地址），检查页面加载、侧边栏、主要功能是否正常。
6. 备用策略：若 `fix.app.py` 未找到或不是 Streamlit：
   - 搜索文件并给出确切路径；
